GB的图像处理：

pixel/pixels    像素
tile/tiles      方块
x shades of gray    x级灰度，GB有四级灰度
Background palette/BG palette   背景调色板
object background palette/OBJ palette   精灵调色板

8*8 pixel tiles     8*8的像素方块
GB的屏幕可以划分为20*18个这样的背景方块，所有用到的背景方块（BG Tiles）都存储在VRAM（GB的VRAM为8KB）中的某一块区域
背景的完全映射叫做背景卷轴（Scrolling），因为可视区域（Viewport）就是在背景的完全映射上移动形成的

Background tile data    背景的方块种数，比如马里奥大陆用了128种方块，因此在屏幕上看到重复方块是常见的
GB中渲染一帧的数据量（只包括背景）：8*8的方块包含的数据量是2Bit*8*8=16Byte，故总的数据量是20*18*16Byte=5.625KiB=5.76KB


window  窗口，窗口可以覆盖背景，可用用来作为分数面板


sprites 精灵，在背景和窗口图层之上的不受形状限制的物体方块（但最多也只能有64个像素），任天堂把它叫做object，GB中有40个精灵，GBA中有36个精灵
所有用到的精灵方块（Sprite Tiles）都存储在VRAM中的某一块区域
精灵的相关数据（OAM记录——记录了精灵的方位信息）存储在OAM RAM中

OAM数据结构：
Position x
Position y
Tile Number
Priority
Filp x
Filp y
Palette


VRAM Memory MAP（GB）
4KB Sprite Tiles: 256 tiles * 16bytes
4KB BG Tiles: 256 tiles * 16bytes
1KB BG Map: 32 * 32 indexes
1KB Window Map: 32 * 32 indexes
1和2有重合部分


GBA Internal Display Memory
05000000-050003FF   BG/OBJ Palette RAM        (1 Kbyte)
05000400-05FFFFFF   Not used
06000000-06017FFF   VRAM - Video RAM          (96 KBytes)
06018000-06FFFFFF   Not used
07000000-070003FF   OAM - OBJ Attributes      (1 Kbyte)
07000400-07FFFFFF   Not used
